# GTA VI — ежедневный дайджест тем

Агент, который раз в день сам собирает до 50 самых обсуждаемых тем по GTA VI
(Reddit, Google News, опционально YouTube), переводит и оформляет их на русском
языке через Claude и публикует страницу на GitHub Pages с переключателем по дням
(каждый день, за который агент отработал, сохраняется в архив и доступен через
выпадающий список вверху страницы).

Пример того, как выглядит страница, был показан в чате отдельным виджетом —
итоговый `docs/index.html` использует тот же стиль.

## Что делает скрипт (`scripts/build_digest.py`)

1. Забирает сырые данные:
   - топ постов за сутки из `r/GTA6` (публичный JSON Reddit, без ключа);
   - последние заголовки по запросу "GTA VI"/"GTA 6" из Google News RSS (без ключа);
   - опционально — статистику просмотров/лайков видео через YouTube Data API
     (нужен бесплатный ключ, см. ниже).
2. Отправляет весь сырой список в Anthropic API (Claude), который убирает дубли,
   группирует упоминания одного события, переводит на русский и оставляет
   ссылки на источники + метрики (просмотры/лайки/апвоуты/комментарии). Выбирает
   до 50 тем (`MAX_TOPICS` в скрипте) — если реальных тем за день меньше, покажет
   столько, сколько реально нашлось, не выдумывая лишние.
3. Сохраняет:
   - `docs/archive/YYYY-MM-DD.json` — темы за конкретный день;
   - `docs/manifest.json` — список всех дат, за которые есть архив (используется
     переключателем дней на странице);
   - `docs/index.html` — статическая страница-оболочка. Сама HTML-разметка не
     меняется каждый день — она через JavaScript (`fetch`) подгружает
     `manifest.json` и нужный `archive/<дата>.json` и рисует карточки на лету,
     поэтому переключение между днями происходит мгновенно, без перезагрузки;
   - `docs/latest.json` — алиас на темы за сегодня, если нужен внешним системам.

## Установка (займёт 10–15 минут)

### 1. Создайте репозиторий на GitHub

Создайте новый **публичный** репозиторий (для бесплатного GitHub Pages) и
загрузите в него содержимое этой папки целиком, сохранив структуру:

```
.github/workflows/daily.yml
scripts/build_digest.py
requirements.txt
docs/
```

### 2. Получите ключ Anthropic API (обязательно)

1. Зайдите на https://console.anthropic.com
2. Создайте API-ключ в разделе API Keys.
3. Пополните баланс на пару долларов — при одном запуске в день расход
   мизерный (один запрос, несколько тысяч токенов).

### 3. Получите ключ YouTube Data API (опционально, но рекомендуется)

Без него агент будет работать только на Reddit + Google News. Чтобы добавить
реальные просмотры/лайки трейлеров и видео:

1. Зайдите в https://console.cloud.google.com
2. Создайте проект → включите "YouTube Data API v3"
3. Создайте API-ключ (Credentials → Create Credentials → API key)
4. Бесплатной квоты (10 000 единиц/день) с большим запасом хватает на один
   запуск в сутки.

### 4. Добавьте ключи как секреты репозитория

В репозитории на GitHub: **Settings → Secrets and variables → Actions → New
repository secret**

- `ANTHROPIC_API_KEY` — ваш ключ Anthropic
- `YOUTUBE_API_KEY` — ваш ключ YouTube (если решили его подключать)

### 5. Включите GitHub Pages

**Settings → Pages → Source**: выберите branch `main`, папку `/docs` → Save.

Через минуту страница будет доступна по адресу вида:
`https://<ваш-логин>.github.io/<имя-репозитория>/`

### 6. Запустите вручную первый раз

Вкладка **Actions → GTA VI Daily Digest → Run workflow**. Дальше агент будет
запускаться сам каждый день в 06:00 UTC (workflow во вкладке Actions также
можно запустить вручную в любой момент).

## Настройка расписания

Измените строку `cron` в `.github/workflows/daily.yml`, например:

```yaml
- cron: "0 9 * * *"   # 09:00 UTC = 12:00 МСК
```

## Настройка источников/количества тем

В `scripts/build_digest.py`:

- `MAX_TOPICS` — сколько тем показывать максимум (по умолчанию 50)
- `fetch_reddit_top(subreddit="GTA6", ...)` — можно добавить ещё сабреддиты
- `fetch_google_news(query=...)` — можно уточнить поисковый запрос
- `ANTHROPIC_MODEL` — можно заменить на более быструю/дешёвую модель

Имейте в виду: реальных уникальных тем по игре за одни сутки обычно меньше
50 (даже в горячие недели — обычно 15–30). Скрипт никогда не выдумывает темы,
чтобы дотянуть до лимита — покажет ровно столько, сколько реально нашлось.

## Переключатель по дням

Каждый запуск агента добавляет новую дату в архив, ничего не перезаписывая.
На странице сверху появляется выпадающий список дат — при выборе даты темы
подгружаются мгновенно (через `fetch('archive/<дата>.json')`), без перезагрузки
страницы. Список дат хранится в `docs/manifest.json` и обновляется автоматически.

Чтобы протестировать переключатель локально до первого реального запуска,
в архиве уже лежит демо-день — откройте `docs/index.html` и переключите дату
в списке.

## Локальный запуск для теста

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...
export YOUTUBE_API_KEY=...   # опционально
python scripts/build_digest.py
```

Затем откройте `docs/index.html` через локальный сервер (просто открыть файл
двойным кликом может не сработать из-за ограничений браузера на `fetch()` для
`file://`):

```bash
cd docs && python3 -m http.server 8000
# откройте http://localhost:8000
```

На GitHub Pages это ограничение не действует — там страница отдаётся по
`https://`, и `fetch()` работает как обычно.
