# GTA VI — топ мемов дня

Агент, который раз в день сам собирает до 50 самых популярных постов с
картинками про GTA VI (Grand Theft Auto VI) с Reddit — не только мемы, но и
фан-арт, находки в трейлерах, спекуляции и другой популярный визуальный
контент. Для каждого поста берётся превью-картинка, ссылка и дата, всё
ранжируется по популярности за день (позиция #1–#50), и публикуется страница
на GitHub Pages с переключателем по дням — каждый день, за который агент
отработал, сохраняется в архив и доступен через выпадающий список вверху
страницы. Новый день в архиве появляется в 00:00 по московскому времени.

**Важное ограничение:** точное число апвоутов/комментариев показать не
получится — источник данных (см. ниже) его не отдаёт без платного или
недоступного сейчас ключа. Вместо цифры показывается позиция в рейтинге
(#1 — самый популярный мем дня по версии Reddit, и так далее).

## Откуда берутся данные

Практичный бесплатный источник картиночных мемов про игру — **Reddit** (в
первую очередь `r/GTA6`, сообщество на несколько миллионов подписчиков, где
постоянно постят мемы про игру).

Путь к этому источнику был не прямым — стоит знать, на что рассчитывать:

1. **OAuth-приложение Reddit Data API** — обычный self-serve способ получить
   ключ (`reddit.com/prefs/apps`). Сейчас (проверено в 2026 году) форма
   создания приложения не даёт завершить регистрацию, а вики Reddit прямо
   говорит, что новые приложения принимаются только по заявке и только при
   наличии "valid moderation use case". Для хобби-агрегатора мемов это не
   подходит.
2. **Imgur API** (`api.imgur.com/oauth2/addclient`) — рассматривали как
   замену. Форма регистрации редиректит на главную страницу, то есть
   self-serve выдача ключей тоже закрыта.
3. **Публичный `reddit.com/*.json` без ключа** — исторически многие хобби-боты
   так и делали. Сейчас возвращает 403 и с обычных IP, и с раннеров GitHub
   Actions — тоже не работает.
4. **Публичная RSS/Atom-лента `reddit.com/r/.../top/.rss`** — единственный
   вариант, который реально ответил 200 и с данными (проверено и локально, и
   через реальный прогон GitHub Actions). Это и используется сейчас.

Минус RSS-подхода: лента не содержит число апвоутов/комментариев — только
заголовок, картинку, ссылку и дату. Зато записи в ленте уже отсортированы
Reddit'ом по популярности за день, поэтому позиция в списке (ранг) — это и
есть сигнал популярности, просто без точной цифры.

Другие источники, которые рассматривались и почему не подошли:
- **Twitter/X API** — платный доступ к поиску по картинкам, не подходит для
  бесплатного ежедневного бота.
- **Tenor/Giphy** — дают числовые метрики (просмотры), но это gif-поисковики
  общего назначения, а не специализированный источник мемов про GTA VI —
  релевантного контента там заметно меньше.
- **Know Your Meme** — нет публичного API, пришлось бы парсить HTML, что
  хрупко и, вероятно, нарушает условия использования сайта.

Список сабреддитов настраивается (`REDDIT_SUBREDDITS`), по умолчанию —
`GTA6,GTA6unmoderated`. Второй сабреддит менее модерируемый, поэтому Claude
на этапе отбора дополнительно отсеивает NSFW/оскорбительный контент оттуда.
Можно добавить другие сабреддиты через запятую, если найдутся ещё активные
мемные сообщества по игре.

## Что делает скрипт (`scripts/build_memes.py`)

1. Забирает топ постов за сутки из указанных сабреддитов через публичную
   RSS/Atom-ленту `reddit.com/r/.../top/.rss` (без ключа), оставляя только
   записи с картинкой-превью — текстовые посты без изображения отбрасываются
   сразу. Позиция поста в ленте (ранг) сохраняется как сигнал популярности.
2. Отправляет посты в Anthropic API (Claude) — **отдельным запросом для каждого
   сабреддита** (не одним общим запросом на все сразу). Так сделано специально:
   при совместной обработке Claude на практике стабильно выбирал посты только
   из `r/GTA6`, полностью игнорируя `r/GTA6unmoderated`, несмотря на прямую
   инструкцию так не делать. Раздельные запросы со своей квотой мест на
   сабреддит (`MAX_MEMES`, поделённое поровну между сабреддитами) гарантируют
   на уровне кода, что каждый источник попадёт в подборку, даже если его посты
   в среднем менее популярны. Каждый запрос:
   - убирает повторы внутри своего сабреддита;
   - отсеивает NSFW/оскорбительный контент и то, что не по теме игры (но НЕ
     отсеивает серьёзные посты только за то, что это не шутка — мемы, арт,
     находки и спекуляции остаются вперемешку);
   - сохраняет исходный порядок по рангу в пределах своей квоты;
   - добавляет развёрнутое описание на русском (4-6 предложений) для каждого поста.
   Результаты по сабреддитам склеиваются в порядке `REDDIT_SUBREDDITS` и
   дедуплицируются по прямой ссылке на картинку (на случай кросс-постов).
3. Сохраняет:
   - `docs/archive/YYYY-MM-DD.json` — мемы за конкретный день;
   - `docs/manifest.json` — список всех дат, за которые есть архив (для
     переключателя дней на странице);
   - `docs/latest.json` — алиас на мемы за сегодня, если нужен внешним системам.
4. `docs/index.html` — статическая страница-оболочка. Разметка не меняется
   каждый день — через JavaScript (`fetch`) подгружается `manifest.json` и
   нужный `archive/<дата>.json`, карточки рисуются на лету, переключение между
   днями происходит мгновенно, без перезагрузки страницы.

## Установка (займёт 15–20 минут)

### 1. Создайте репозиторий на GitHub

Создайте новый **публичный** репозиторий (для бесплатного GitHub Pages) и
загрузите в него содержимое этой папки целиком, сохранив структуру:

```
.github/workflows/daily.yml
scripts/build_memes.py
requirements.txt
docs/
```

### 2. Получите ключ Anthropic API (обязательно)

1. Зайдите на https://console.anthropic.com
2. Создайте API-ключ в разделе API Keys.
3. Пополните баланс на пару долларов — при одном запуске в день расход
   мизерный (один запрос, несколько тысяч токенов).

### 3. Добавьте ключ как секрет репозитория

В репозитории на GitHub: **Settings → Secrets and variables → Actions → New
repository secret**

- `ANTHROPIC_API_KEY` — ваш ключ Anthropic

Опционально, во вкладке **Variables** там же можно добавить `REDDIT_SUBREDDITS`
(например `GTA6,gtavi`), если решите добавить ещё сабреддиты. Ключи Reddit/Imgur
не нужны — сбор идёт через публичный `reddit.com/*.json` (см. раздел "Откуда
берутся данные" выше про ограничения этого подхода).

### 4. Включите GitHub Pages

**Settings → Pages → Source**: выберите branch `main`, папку `/docs` → Save.

Через минуту страница будет доступна по адресу вида:
`https://<ваш-логин>.github.io/<имя-репозитория>/`

### 5. Запустите вручную первый раз

Вкладка **Actions → GTA VI Daily Memes → Run workflow**. Дальше агент будет
запускаться сам каждый день в 21:00 UTC (00:00 МСК) — новый день появляется в
выпадающем списке на странице ровно в полночь по московскому времени (workflow
во вкладке Actions также можно запустить вручную в любой момент).

## Настройка расписания

Измените строку `cron` в `.github/workflows/daily.yml`, например:

```yaml
- cron: "0 21 * * *"   # 21:00 UTC = 00:00 МСК
```

Дата архивного файла считается по московскому времени (`MSK_TZ` в
`scripts/build_memes.py`), а не по UTC — если поменяете расписание на другой
часовой пояс, поправьте и эту константу, иначе новый день будет появляться не
в полночь, а со сдвигом.

## Настройка количества постов/сабреддитов

В `scripts/build_memes.py`:

- `MAX_MEMES` — сколько постов показывать максимум (по умолчанию 50)
- `REDDIT_SUBREDDITS` (переменная окружения) — список сабреддитов через запятую

Имейте в виду: реальных популярных постов за одни сутки на `r/GTA6` и
`r/GTA6unmoderated` обычно меньше 50 в спокойные дни и заметно больше в дни
выхода трейлеров/новостей. Скрипт никогда не выдумывает посты, чтобы
дотянуть до лимита.

## Переключатель по дням

Каждый запуск агента добавляет новую дату в архив, ничего не перезаписывая.
На странице сверху появляется выпадающий список дат — при выборе даты мемы
подгружаются мгновенно (через `fetch('archive/<дата>.json')`), без перезагрузки
страницы. Список дат хранится в `docs/manifest.json` и обновляется автоматически.

Чтобы протестировать переключатель локально до первого реального запуска, в
архиве уже лежит демо-день (`docs/archive/2026-07-10.json`, картинки —
плейсхолдеры) — откройте страницу и убедитесь, что сетка карточек рисуется
корректно. После первого реального запуска скрипта демо-день можно удалить.

## Локальный запуск для теста

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...
python scripts/build_memes.py
```

Затем откройте `docs/index.html` через локальный сервер (просто открыть файл
двойным кликом может не сработать из-за ограничений браузера на `fetch()` для
`file://`):

```bash
cd docs && python3 -m http.server 8000
# откройте http://localhost:8000
```

На GitHub Pages это ограничение не действует — там страница отдаётся по
`https://`, и `fetch()` работает как обычно.
