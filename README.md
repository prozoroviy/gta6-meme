# GTA VI — топ мемов дня

Агент, который раз в день сам собирает до 50 самых популярных и смешных мемов
про GTA VI (Grand Theft Auto VI), берёт для каждого превью-картинку, ссылку,
дату и число реакций (апвоуты/комментарии), и публикует страницу на GitHub
Pages с переключателем по дням — каждый день, за который агент отработал,
сохраняется в архив и доступен через выпадающий список вверху страницы.

## Откуда берутся данные

Практичный бесплатный источник картиночных мемов со счётчиком реакций и
датой публикации — **Reddit** (в первую очередь `r/GTA6`, сообщество на
несколько миллионов подписчиков, где постоянно постят мемы про игру).

Скрипт использует публичный `reddit.com/*.json` **без ключа**. Это осознанный
выбор, а не первый вариант: официальная регистрация OAuth-приложений для
Reddit Data API сейчас (по состоянию на 2026 год) закрыта для новых
хобби-проектов — форма создания приложения на `reddit.com/prefs/apps`
блокирует отправку, а вики Reddit прямо говорит, что новые приложения
принимаются только по заявке и только при наличии "valid moderation use
case". Проверили и альтернативу — самостоятельную регистрацию ключа Imgur
(`api.imgur.com/oauth2/addclient`) — форма тоже недоступна (редиректит на
главную), похоже, тоже закрыта для новых разработчиков.

Из-за этого публичный JSON без ключа — единственный рабочий вариант сейчас.
Минус: это недокументированный эндпоинт, который иногда возвращает 403 для
подозрительных запросов (в основном для дата-центровых IP без признаков
браузера). С реальных пользовательских машин и раннеров GitHub Actions он в
большинстве случаев отвечает нормально, но полной гарантии стабильности нет —
если ежедневный workflow вдруг перестанет получать данные, скорее всего
Reddit ужесточил блокировку, и тогда стоит будет вернуться к вопросу ключа.

Другие источники, которые рассматривались и почему не подошли:
- **Twitter/X API** — платный доступ к поиску по картинкам, не подходит для
  бесплатного ежедневного бота.
- **Imgur API** — см. выше, self-serve регистрация ключа сейчас недоступна.
- **Know Your Meme** — нет публичного API, пришлось бы парсить HTML, что
  хрупко и, вероятно, нарушает условия использования сайта.

Список сабреддитов настраивается (`REDDIT_SUBREDDITS`), по умолчанию — только
`GTA6`. Можно добавить другие через запятую, если найдутся ещё активные
мемные сообщества по игре.

## Что делает скрипт (`scripts/build_memes.py`)

1. Забирает топ постов за сутки из указанных сабреддитов через публичный
   `reddit.com/*.json` (без ключа), оставляя только визуальный контент
   (картинки, гифки, галереи, видео) — текстовые посты и ссылки на статьи
   отбрасываются сразу.
2. Отправляет весь список сырых постов в Anthropic API (Claude), который:
   - убирает повторы (репосты одной картинки под разными заголовками);
   - отсеивает то, что не мем (скриншоты без шутки, арт, серьёзные посты);
   - сортирует по популярности/смешности и оставляет не более 50 (`MAX_MEMES`);
   - добавляет короткую смешную подпись на русском для каждого мема.
3. Сохраняет:
   - `docs/archive/YYYY-MM-DD.json` — мемы за конкретный день;
   - `docs/manifest.json` — список всех дат, за которые есть архив (для
     переключателя дней на странице);
   - `docs/latest.json` — алиас на мемы за сегодня, если нужен внешним системам.
4. `docs/index.html` — статическая страница-оболочка. Разметка не меняется
   каждый день — через JavaScript (`fetch`) подгружается `manifest.json` и
   нужный `archive/<дата>.json`, карточки рисуются на лету, переключение между
   днями происходит мгновенно, без перезагрузки страницы.

## Установка (займёт 15–20 минут)

### 1. Создайте репозиторий на GitHub

Создайте новый **публичный** репозиторий (для бесплатного GitHub Pages) и
загрузите в него содержимое этой папки целиком, сохранив структуру:

```
.github/workflows/daily.yml
scripts/build_memes.py
requirements.txt
docs/
```

### 2. Получите ключ Anthropic API (обязательно)

1. Зайдите на https://console.anthropic.com
2. Создайте API-ключ в разделе API Keys.
3. Пополните баланс на пару долларов — при одном запуске в день расход
   мизерный (один запрос, несколько тысяч токенов).

### 3. Добавьте ключ как секрет репозитория

В репозитории на GitHub: **Settings → Secrets and variables → Actions → New
repository secret**

- `ANTHROPIC_API_KEY` — ваш ключ Anthropic

Опционально, во вкладке **Variables** там же можно добавить `REDDIT_SUBREDDITS`
(например `GTA6,gtavi`), если решите добавить ещё сабреддиты. Ключи Reddit/Imgur
не нужны — сбор идёт через публичный `reddit.com/*.json` (см. раздел "Откуда
берутся данные" выше про ограничения этого подхода).

### 4. Включите GitHub Pages

**Settings → Pages → Source**: выберите branch `main`, папку `/docs` → Save.

Через минуту страница будет доступна по адресу вида:
`https://<ваш-логин>.github.io/<имя-репозитория>/`

### 5. Запустите вручную первый раз

Вкладка **Actions → GTA VI Daily Memes → Run workflow**. Дальше агент будет
запускаться сам каждый день в 07:00 UTC (workflow во вкладке Actions также
можно запустить вручную в любой момент).

## Настройка расписания

Измените строку `cron` в `.github/workflows/daily.yml`, например:

```yaml
- cron: "0 9 * * *"   # 09:00 UTC = 12:00 МСК
```

## Настройка количества мемов/сабреддитов

В `scripts/build_memes.py`:

- `MAX_MEMES` — сколько мемов показывать максимум (по умолчанию 50)
- `REDDIT_SUBREDDITS` (переменная окружения) — список сабреддитов через запятую

Имейте в виду: реальных смешных мемов за одни сутки на `r/GTA6` обычно
меньше 50 в спокойные дни и заметно больше в дни выхода трейлеров/новостей.
Скрипт никогда не выдумывает мемы, чтобы дотянуть до лимита.

## Переключатель по дням

Каждый запуск агента добавляет новую дату в архив, ничего не перезаписывая.
На странице сверху появляется выпадающий список дат — при выборе даты мемы
подгружаются мгновенно (через `fetch('archive/<дата>.json')`), без перезагрузки
страницы. Список дат хранится в `docs/manifest.json` и обновляется автоматически.

Чтобы протестировать переключатель локально до первого реального запуска, в
архиве уже лежит демо-день (`docs/archive/2026-07-10.json`, картинки —
плейсхолдеры) — откройте страницу и убедитесь, что сетка карточек рисуется
корректно. После первого реального запуска скрипта демо-день можно удалить.

## Локальный запуск для теста

```bash
pip install -r requirements.txt
export ANTHROPIC_API_KEY=sk-ant-...
python scripts/build_memes.py
```

Затем откройте `docs/index.html` через локальный сервер (просто открыть файл
двойным кликом может не сработать из-за ограничений браузера на `fetch()` для
`file://`):

```bash
cd docs && python3 -m http.server 8000
# откройте http://localhost:8000
```

На GitHub Pages это ограничение не действует — там страница отдаётся по
`https://`, и `fetch()` работает как обычно.
